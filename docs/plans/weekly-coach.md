# Weekly Coach (Layer 4) Implementation Plan

> **For agentic workers:** Implement task-by-task with the `subagent-driven-development` skill. Steps use checkbox (`- [ ]`). Phase gates are **hard checkpoints** — do NOT advance past a gate that has not passed.

**Goal:** End-of-evening Day 4 deliverable per `docs/build-plan.md` Day 4 Layer 4: a "This week's insight" panel in the WakeProof home view displays a tasteful coaching insight Claude Opus 4.7 produced from 14 days of seed wake-history + the Layer 2 memory profile in a single 1M-context call. The live call runs ONCE from the dev laptop during Phase B; the output JSON is committed to the repo and loaded by iOS as a static resource. This is the documented "mocked for demo" path from `docs/opus-4-7-strategy.md` Layer 4.

**Architecture:** Three-phase. **Phase A** lands seed data, the 1M-context prompt template, a Python dev-script that runs the live call, a Swift `WeeklyCoach` service that loads the output JSON, a `WeeklyInsightView` SwiftUI panel, and tests. All iOS code consumes a committed static JSON; tests use a hand-authored fixture (never the live output). **Phase B** runs the real 1M-context call ONCE from the dev laptop against the Anthropic API directly (bypasses the Vercel proxy because the call's wall-clock will exceed the Vercel Hobby 10 s cap and because the single-call pattern has no reason to go through a proxy that adds no value). The live output JSON is then committed and wired into the app. **Phase C** runs the review pipeline. Because Layer 4 is the smallest surface area in the four-layer stack, this plan is lighter than the Layer 2 / Layer 3 plans; expect ~800 lines vs. ~2000.

**Tech Stack:** Swift + SwiftUI (iOS 17+). Python 3 for the dev script (Anthropic Python SDK installed via `pip`). JSON on-disk as the handoff. No new iOS SPM dependencies. `Logger(subsystem: "com.wakeproof.coach", …)`. No proxy changes, no new BGProcessingTask registration, no new permissions.

**Non-goals for this plan (deferred):**
- Live weekly scheduling (cron-style generation from an always-on server) — post-hackathon
- User-facing data that isn't the seeded history (collecting 14 real wake days requires 14 real days)
- Multi-week comparison (this week vs. last week) — post-hackathon
- Embedding the insight into the alarm ringing UX (e.g., "you're falling behind on your contract") — Day 5 polish if we have UX time
- Layer 1 prompt changes to mention the coach's insight — out of scope
- Managed-Agent consumption of the coach's output — Layer 3 is self-contained

## Architecture decision — dev-laptop generation, committed output

The strategy doc (`docs/opus-4-7-strategy.md` Layer 4) and build plan (`docs/build-plan.md` Day 4) both describe Layer 4 as **mocked for demo, not a live scheduled job**. Three implementation options were considered during planning brainstorm:

1. **Dev-laptop generation + committed output (chosen).** Run the 1M-context call once from the dev machine using a direct Anthropic key. Save output to `WakeProof/Resources/weekly-insight-seed.json`. iOS loads the static file. Cost: $3–5 per generation. Iteration cost: same. Zero infra changes.
2. **Vercel Pro upgrade ($20/mo, 60s cap).** Would let the iOS app call a new `/v1/messages` with a long-context request synchronously. Adds a real subscription cost outside the hackathon credit budget. Rejected on cost.
3. **Move Layer 4 to Fly.io / Railway (free tier).** Longer request budgets on free hosts, but adds infrastructure on Day 4 that has no other WakeProof consumer. Rejected on scope.

Option 1 is the cleanest fit with the "mocked for demo" language already in the build plan. Demo narrative: *"This insight was generated by Opus 4.7's 1M-context window — the full 14 days of wake data, plus this user's memory profile, produced this single paragraph of insight."* The user sees a real Opus 4.7 output; it is not synthetic.

Honesty note for the demo: the seed data is synthetic (clearly labeled in `docs/weekly-coach-seed.md`). The INSIGHT is real, produced by a real 1M-context call against real Claude. Judges who look at the repo will see both the synthetic inputs and the real output — the README section on Layer 4 should explain this up front.

## Critical constraints (Day 4, layer 4)

- **DO NOT** modify Day 3, Layer 2, or Layer 3 code except the one-line wiring in `WakeProofApp` that mounts `WeeklyCoach` and shows `WeeklyInsightView` on the home screen.
- **DO NOT** hardcode an Anthropic key anywhere on the iOS side. The 1M-context call is a dev-machine operation; the key lives in a local environment variable on the dev laptop.
- **DO NOT** commit the Anthropic API key. The Python script reads from `$ANTHROPIC_API_KEY`; the script must refuse to run without the env var. If a user pastes the key into the script during edits, pre-commit check in `.pre-commit-config.yaml` (if present) or a grep catches it.
- **DO NOT** commit the response JSON before inspecting it for sensitive content. The seed data is synthetic, but the live call could incorporate incidental content; always read the JSON before `git add`.
- **DO NOT** iterate on the prompt more than 3 times live without user confirmation. Each iteration is ~$3–5 of credit.
- **DO NOT** change the Vercel proxy at all for this plan. Layer 4's call is dev-laptop-direct.
- **DO NOT** break the existing "you woke up" UI flow. The `WeeklyInsightView` is an additive panel on `AlarmSchedulerView`, not a replacement.
- **DO NOT** run `git push`. Local commits only.

```markdown
# Weekly Coach Seed Data

> `WakeProof/WakeProof/Resources/mock-wake-history-seed.json` holds 14 days of **synthetic** wake-attempt data. It is NOT a record of any real user.
>
> The data carries one intentional pattern: Monday mornings (2026-04-13 and 2026-04-20) show RETRY-heavy flows relative to the rest of the week. The Layer 4 insight-generator's job is to surface this pattern in one or two tasteful sentences.

## Why synthetic

A 5-day hackathon produces at most 4 real wake days — not enough to show a weekly insight landing. Synthetic data lets Layer 4 demonstrate the 1M-context synthesis mechanic without needing 14 real mornings from the user.

## Demo-day honesty

In the demo video and submission README, make it clear: "The 14-day history is seed data. The INSIGHT you see on-screen was generated by a real Opus 4.7 call against that seed + this user's real memory profile." Both facts are true and the judges can verify by reading the repo.

## Regeneration semantics

If the seed is changed, re-run `scripts/generate-weekly-insight.py` so the committed insight JSON matches. The script writes a `seedChecksum` field iOS can sanity-check against the loaded seed file.

## Fields

- `day`: human-friendly date + day-of-week label
- `wakeTime`: local clock time of the alarm fire
- `verdictTrail`: ordered list of verdicts for that morning's attempts
- `retries`: total retries in this morning
- `confidence`: Claude's final confidence score on the terminal verdict
- `note`: one-line Claude-authored note (mimics `memory_update.history_note` style)
```

```python
#!/usr/bin/env python3
"""Generate WakeProof's weekly coach insight (Layer 4).

Reads the 14-day synthetic seed + the user's memory profile, calls Claude Opus 4.7
with the full context, and writes the result as WakeProof/Resources/weekly-insight-seed.json.

Run once per seed change. Expects ANTHROPIC_API_KEY in the environment. Refuses to
run without it.
"""
from __future__ import annotations

import argparse
import hashlib
import json
import os
import sys
import time
from datetime import datetime, timezone
from pathlib import Path

try:
    import anthropic
except ImportError:
    sys.exit("Install the Anthropic Python SDK first: pip install anthropic")


SYSTEM_PROMPT = """You are the weekly coach of WakeProof, a wake-up accountability app. The user will give you:
1. Their last 14 days of wake-attempt history (a JSON array — synthetic for demo purposes but structurally real).
2. Their persistent memory profile (markdown; may be empty).

Your task: produce a single piece of coaching insight in 2–4 sentences. Warm, specific, concrete — not platitudes. Your insight should be actionable or acknowledge a pattern the user might not have noticed.

Rules:
- Use the full 14 days. Do not speculate beyond the window.
- Do not speculate medically.
- If you identify a pattern (e.g., "Mondays are harder than Tuesdays"), say so and name what could help — earlier bedtime on Sunday, etc.
- If the memory profile contradicts the 14-day data, trust the 14-day data and mention the discrepancy gently.
- Output a single JSON object:

{
  "insightText": "2–4 sentences of plain prose, no markdown",
  "patternNoticed": "short name of the pattern if any, or null",
  "suggestedAction": "one short sentence, or null"
}

No text outside the JSON. No markdown fences.
"""


def build_user_prompt(history: dict, profile: str) -> str:
    return f"""<wake_history>
{json.dumps(history["entries"], indent=2)}
</wake_history>

<memory_profile>
{profile if profile.strip() else "(empty)"}
</memory_profile>

Produce the coaching insight JSON now."""


def main() -> int:
    parser = argparse.ArgumentParser()
    parser.add_argument("--history", default="WakeProof/WakeProof/Resources/mock-wake-history-seed.json")
    parser.add_argument("--profile", default="", help="Path to a profile.md file, or empty string for no profile")
    parser.add_argument("--out", default="WakeProof/WakeProof/Resources/weekly-insight-seed.json")
    parser.add_argument("--model", default="claude-opus-4-7")
    parser.add_argument("--dry-run", action="store_true", help="Print the prompt but do not call the API")
    args = parser.parse_args()

    if not os.environ.get("ANTHROPIC_API_KEY"):
        sys.exit("ANTHROPIC_API_KEY not set in env. Refusing to run without it.")

    history_path = Path(args.history)
    if not history_path.exists():
        sys.exit(f"History file not found: {history_path}")

    history = json.loads(history_path.read_text())
    profile_text = ""
    if args.profile:
        profile_path = Path(args.profile)
        if profile_path.exists():
            profile_text = profile_path.read_text()

    user_prompt = build_user_prompt(history, profile_text)

    print(f"System prompt bytes: {len(SYSTEM_PROMPT.encode('utf-8'))}")
    print(f"User prompt bytes:   {len(user_prompt.encode('utf-8'))}")

    if args.dry_run:
        print("\n--- User prompt ---\n")
        print(user_prompt)
        return 0

    client = anthropic.Anthropic()
    t0 = time.time()
    response = client.messages.create(
        model=args.model,
        max_tokens=1024,
        system=SYSTEM_PROMPT,
        messages=[{"role": "user", "content": user_prompt}],
    )
    elapsed = time.time() - t0

    content_text = "".join(
        block.text for block in response.content if getattr(block, "type", None) == "text"
    )
    # Claude is instructed to emit a single JSON object; parse it defensively.
    try:
        payload = json.loads(content_text)
    except json.JSONDecodeError as exc:
        sys.exit(f"Claude output did not parse as JSON: {exc}\nRaw:\n{content_text}")

    seed_checksum = hashlib.sha256(history_path.read_bytes()).hexdigest()[:16]

    wrapped = {
        "generatedAt": datetime.now(timezone.utc).replace(microsecond=0).isoformat(),
        "model": args.model,
        "elapsedSeconds": round(elapsed, 2),
        "inputTokens": response.usage.input_tokens,
        "outputTokens": response.usage.output_tokens,
        "seedChecksum": seed_checksum,
        "insight": payload,
    }

    out_path = Path(args.out)
    out_path.parent.mkdir(parents=True, exist_ok=True)
    out_path.write_text(json.dumps(wrapped, indent=2, ensure_ascii=False) + "\n")
    print(f"\nWrote {out_path} ({len(wrapped['insight']['insightText'])} chars of insight)")
    print(f"Elapsed: {elapsed:.2f}s, tokens in/out: {response.usage.input_tokens}/{response.usage.output_tokens}")
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

```swift
//
//  WeeklyCoach.swift
//  WakeProof
//
//  Loads the committed weekly-insight-seed.json (produced by the dev-laptop
//  generation script) and exposes its parsed contents. Read-only at runtime;
//  regeneration is a dev-machine operation.
//

import Foundation
import Observation
import os

@Observable
@MainActor
final class WeeklyCoach {

    struct Insight: Codable, Equatable {
        let insightText: String
        let patternNoticed: String?
        let suggestedAction: String?
    }

    struct Wrapper: Codable, Equatable {
        let generatedAt: Date
        let model: String
        let elapsedSeconds: Double
        let inputTokens: Int
        let outputTokens: Int
        let seedChecksum: String
        let insight: Insight
    }

    private(set) var wrapper: Wrapper?

    private let logger = Logger(subsystem: "com.wakeproof.coach", category: "weekly")

    /// Injectable for tests — production uses the bundle resource.
    init(resourceURL: URL? = Self.defaultResourceURL) {
        guard let resourceURL else {
            logger.info("WeeklyCoach: no resource URL, running without an insight")
            return
        }
        do {
            let data = try Data(contentsOf: resourceURL)
            let decoder = JSONDecoder()
            decoder.dateDecodingStrategy = .iso8601
            self.wrapper = try decoder.decode(Wrapper.self, from: data)
            logger.info("WeeklyCoach loaded insight generated=\(self.wrapper?.generatedAt.ISO8601Format() ?? "?", privacy: .public) tokens=\(self.wrapper?.inputTokens ?? 0, privacy: .public)/\(self.wrapper?.outputTokens ?? 0, privacy: .public)")
        } catch {
            logger.error("WeeklyCoach failed to load insight: \(error.localizedDescription, privacy: .public)")
        }
    }

    var currentInsight: Insight? { wrapper?.insight }

    var generatedAt: Date? { wrapper?.generatedAt }

    private static var defaultResourceURL: URL? {
        Bundle.main.url(forResource: "weekly-insight-seed", withExtension: "json")
    }
}
```

| Condition | Action |
|---|---|
